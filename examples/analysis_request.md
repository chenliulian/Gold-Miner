# GoldMiner 使用示例

## 示例 1：查询广告消耗数据

**用户输入：**
```
查询昨天各广告账户的消耗情况，按消耗降序排列
```

**Agent 执行：**
1. 自动匹配表：`mi_ads_dmp.dwd_ew_ads_show_res_clk_dld_conv_hi`
2. 生成并执行 SQL
3. 返回结果报告

---

## 示例 2：分析贷款 CVR 模型

**用户输入：**
```
分析贷款CVR模型的训练数据，统计各标签的样本分布
```

**Agent 执行：**
1. 自动匹配表：`mi_ads_dmp.dwd_dld_loancvr_model_train_data_di`
2. 统计 label、label_active、label_register 分布
3. 生成分析报告

---

## 示例 3：查询创意素材

**用户输入：**
```
查询昨天新增的创意素材，按广告主分组
```

**Agent 执行：**
1. 自动匹配表：`com_cdm.dim_creativity_dd`
2. 生成查询 SQL
3. 返回创意列表

---

## 示例 4：计算 CTR/CVR

**用户输入：**
```
计算昨天各广告计划的CTR和CVR
```

**Agent 执行：**
1. 匹配明细表：`mi_ads_dmp.dwd_ew_ads_show_res_clk_dld_conv_hi`
2. 计算点击率和转化率
3. 输出排名结果

---

## 示例 5：多轮对话分析

**第一轮：**
```
> 查询上周的消耗趋势
[Agent 返回每日消耗数据]
```

**第二轮：**
```
> 找出消耗最高的那一天，分析各账户的分布
[Agent 基于上下文，分析峰值日账户分布]
```

**第三轮：**
```
> 对比前一周的数据
[Agent 保持对话上下文，生成对比分析]
```

---

## 常用命令

```
/cancel  - 取消当前正在执行的 SQL 任务
/reset   - 清空对话历史，开始新会话
quit     - 退出 CLI
```

## 提示

- 无需指定表名，Agent 会根据问题自动匹配
- 支持自然语言描述，如"昨天"、"上周"、"最近7天"等
- 对话会保持上下文，可以进行追问和深入分析
