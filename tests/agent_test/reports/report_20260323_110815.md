## 关键发现

### 1. 消耗数据异常
- **两周消耗均为 0**，但展示和点击数据正常
- 可能原因：
  - 该广告主使用其他计费方式（如 CPD/CPI），消耗记录在其他字段
  - 数据同步延迟或计费数据未回传
  - 广告主享受免费流量或测试账户

### 2. 流量大幅增长
- **展示数增长 150%**：从 58.4 万 → 145.8 万
- **点击数增长 106%**：从 2.7 万 → 5.6 万
- 说明广告投放力度显著加大

### 3. CTR 有所下降
- **CTR 从 4.64% 降至 3.83%**，下降 0.81 个百分点
- 可能原因：
  - 流量扩大导致受众精准度下降
  - 新增流量质量较低
  - 创意疲劳或竞争加剧

## 建议

1. **排查消耗数据**：检查 `cpm_cost`、`cpc_cost`、`cpd_cost`、`cpi_cost` 等各计费类型字段，确认实际扣费记录

## 数据说明
- 数据来源：`com_cdm.dws_tracker_ad_cpc_cost_hi`
- 消耗计算：`cpm_cost + cpc_cost`（表中已有美元计价字段）
- CTR 计算：`SUM(click_cnt) / SUM(show_cnt)`

```sql
SELECT 
    CASE 
        WHEN SUBSTR(dh, 1, 8) BETWEEN '20260302' AND '20260308' THEN '上上周 (0302-0308)'
        WHEN SUBSTR(dh, 1, 8) BETWEEN '20260309' AND '20260315' THEN '上周 (0309-0315)'
        ELSE '其他'
    END as week_period,
    SUM(cpm_cost + cpc_cost) as cost_usd,
    SUM(show_cnt) as show_cnt,
    SUM(click_cnt) as click_cnt,
    ROUND(SUM(click_cnt) * 100.0 / NULLIF(SUM(show_cnt), 0), 4) as ctr_pct,
    ROUND(SUM(cpm_cost + cpc_cost) / NULLIF(SUM(click_cnt), 0), 4) as cpc_usd
FROM com_cdm.dws_tracker_ad_cpc_cost_hi
WHERE dh >= '2026030200'
AND dh <= '2026031523'
AND advertiser_id = '2368'
GROUP BY 
    CASE 
        WHEN SUBSTR(dh, 1, 8) BETWEEN '20260302' AND '20260308' THEN '上上周 (0302-0308)'
        WHEN SUBSTR(dh, 1, 8) BETWEEN '20260309' AND '20260315' THEN '上周 (0309-0315)'
        ELSE '其他'
    END
ORDER BY week_period
LIMIT 100
```
